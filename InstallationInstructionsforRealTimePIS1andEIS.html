1.1.1 Installation: Instructions for Integration Server with JDBC Adapter[1]

1.1.1.1 Section I

Using a pre-prepared installation script
  1. Go to E:\ProductImage
  2. Open the SoftwareAG Installer

                

  1. Click the Advanced Options
  2. Navigate to the Scripts Tab

Note: This guide assumes that ”E:\ProductImage folder” preexists/copied to the target installation server.  The installation Image path E:\ProductImage\1011_Image_10212022.zip

  1. Select the Script from the path E:\InstallationHelper\IS_Script<ServerName>.txt[2]
  2. Click ok, and is followed by the below screen to confirm the execution of the installation script[3].

        

  1. Click Next to continue

  1. The Installation process could take about ten minutes or more to complete.

  1. Once the installation is complete, proceed to Section II

1.1.1.2 Section II

This section involves the following steps

  1. Installing the windows services.
  2. Update the logon user and password for the services

Step A: Installing the windows Services

Software AG by default registers the service “SoftwareAG Platform Manager” and “Software AG webMethods Integration Server 10.11”

Step B: Update the logon user and password for the services
  1. Go to “services”
  2. Select the service “Software AG webMethods Integration Server 10.11” and right click
  3. Navigate to the “Logon” tab
  4. Select the radio button “This account”

  1. Type the username and password for the designated server and click ok.
  2. Repeat the same steps for “Software AG Platform Manager”

  1. For services like “Software AG Platform Manager”, which are currently running, when you apply the Logon credentials, you will receive a prompt mentioning “The new logon name will not take effect until you stop and restart the service”.

  1. The above step completes the installation process for “Software AG webMethods Integration Server 10.11"
  2. After completing the above steps please include the following screen shot in the installation capture document, and attach to the JIRA Task.

        

1.1.1.3 Section III

This section provides the instructions for running “Software AG Update Manager” using the prebuilt image.

Step A: Installing the Fixes using Software AG Upgrade Manager
  1. Go to E:\SoftwareAG\SAGUpdateManager\bin
  2. Open the SAGUpdateManagerGUI and Click Manage Fixes

  1. Click  Install fixes from image

  1. Select the image file “E:\ProductImage\FixImage11_1207”

  1. Click Next to continue

  1. Select all the Fixes and Click Next to continue

  1. Software AG Update Manager lists the names of the packages that have any pre or post requisite installation steps.

  1. Click the Read Me (<- This step is not required, since this has already been verified).
  2. Click the check box, “Pre-installation steps have been completed” and click Next to continue

  1. Click OK to the prompt asking to stop “Integration Server” and “My webMethods Server”[4]

        

  1. Click Next to continue

        

  1. This process can take anywhere from ten minutes to thirty minutes.

  1.  Click the readme document for Integration Server Tanuki 10.11 Fix 3

  1. This step completes the first part of the fix installation.

Step B: Post Installation Steps for Integration Server Tanuki 10.11 Fix 1

  1. The Fix “Integration Server Tanuki 10.11 Fix 1” requires eight post installation steps.
  2. Go to  E:\SoftwareAG\profiles\IS_default\configuration\wrapper.conf
  3. Take a backup of the current file as “wrapper_original.conf”
  4. Open the file  E:\SoftwareAG\profiles\IS_default\configuration\wrapper.conf
  1. Update the following

wrapper.java.classpath.1=E:\SoftwareAG\common\lib\tw-3.5.43 with

 wrapper.java.classpath.1=E:\SoftwareAG\common\lib\tw-3.5.48

  1. Save the changes and close the file.

  1. Save Changes and close the file.
  2. Go to the folder E:\SoftwareAG\profiles\IS_default\bin
  3. Take a backup of the current file as “service.bat” and “setenv.bat”
  4. Open the file E:\SoftwareAG\profiles\IS_default\bin\service.bat
  1. Update the following

set _WRAPPER_BASE=wrapper--3.5.43.exe

with

set _WRAPPER_BASE=wrapper--3.5.48.exe

        Save the changes and close the file.

  1. Open the file E:\SoftwareAG\profiles\IS_default\bin\setenv.bat

        Update the following

set WRAPPER_BASE=”E:\SoftwareAG\common\bin\wrapper-3.5.43.exe

with

set WRAPPER_BASE=”E:\SoftwareAG\common\bin\wrapper-3.5.48.exe

Save the changes and close the file.

  1. This step completes the post installation process for the fix update “Integration Server Tanuki 1011 Fix 1 (w64)”
